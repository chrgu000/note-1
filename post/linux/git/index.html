2.1. 命令

2.1.1. 远程分支

  1. 查看远程分支

    $ git remote -v
origin  git@git.oschina.net:dishui/local_b2b.git (fetch)
origin  git@git.oschina.net:dishui/local_b2b.git (push)

  2. 添加远程分支

    git remote add origin git@git.oschina.net:dishui/local_b2b.git

  3. push 本地分支到远程分支(如果没有,会自动创建)

    git push origin bigdata #bigdata 分支名称

  4. 设置远程分支

    git remote set-url origin git@github.com:dishuiGit/SublimeNote.git

  5. 添加

    git add .		#添加
git reset .		#撤销添加

  6. 更新

git pull origin master

2.2. 本地目录(存在文件),上传到码云

# cd <dir>
git init .
vi .gitignore # 设置不需要提交的文件
git add . && git commit -m "msg"
git branch --set-upstream-to=origin/<branch> master # 本地分支与远程分支关联
git pull
git push
git remote add origin git@git.oschina.net:dishui/project.git
git remote set-url origin git@git.oschina.net:dishui/project.git
git add . && git commit -m "2017-07-10"
git push origin master:pingan

2.2.2. Q & A

  1. error: src refspec bigdata does not match any.

    [root@localhost dockerfiles]# git push origin bigdata
error: src refspec bigdata does not match any.
error: 无法推送一些引用到 'git@git.oschina.net:dishui/dockerfiles.git'
    解决
    git push origin master:bigdata
